1Background

What is the Move In Module?

Move In is the primary onboarding workflow in TFW — the moment a new tenant is formally assigned a storage unit and a financial relationship begins. The module handles three parallel modes (Move In, Estimate, and Reservation), manages complex billing logic for anniversary and non-anniversary facilities, drives configurable document generation, and integrates e-signature delivery — all in a single, stateful multi-step flow accessible from seven different entry points in the platform.

4Three Operating Modes

Move In, Estimate & Reservation

The module runs three modes from the same UI surface — with full cross-mode state retention where data is compatible, and clear data-loss warnings with user confirmation when it isn't.

Mode 01 · Primary

Move In

Full rental commitment — assigns a unit to a tenant and creates charges on the ledger. Requires unit number, tenant, and ledger. Supports promo plans, insurance, and full document delivery.

  • Unit + tenant + ledger required
  • All charges generated
  • Lease signed at this step
  • Promo & insurance supported
  • Anniversary sync billing available
Mode 02 · Pre-sale

Estimate

Generates a cost estimate for a prospective tenant — no financial commitment. Tenant or lead can be selected; ledger selection is not mandatory. Can seamlessly convert to Move In or Reservation.

  • Works with leads (no account needed)
  • No charges created
  • Ledger selection optional
  • Estimate report generated
  • Converts to Move In preserving data
Mode 03 · Hold

Reservation

Holds a unit type for a prospective tenant with a reservation code — without assigning a specific unit. Insurance and promo plan data is cleared on switch (with user warning). Converts to Move In on arrival.

  • Unit type only (no unit number)
  • Reservation code issued
  • No promo/insurance (warned on switch)
  • Overbook support per unit type
  • Tenant notification on reservation
5Lease & E-Sign

Legal document delivery flows

The lease is a legal occupancy agreement auto-populated with tenant and unit data at move-in time. Three delivery methods are supported — each with distinct UX, status management, and post-move-in re-delivery options.

🖨️

Print

The default delivery method. The lease is printed at move-in time, and it is assumed the tenant signs it in person. Move-in completes immediately and the unit status moves to Rented. Requires the Syrasoft Cloud Assist app to be configured.

Must Have · Default
✉️

Email for E-Sign

An email with the lease is automatically sent to the tenant's email after move-in. The unit status is set to Esign Pending until the tenant signs via the tenant-facing portal. Document validity is 7 days. Users can manually update status between Rented and Esign Pending. Requires email service to be configured and tenant email to be enabled.

Must Have · Async
📲

Send to Device

The lease is pushed to a connected device running the Syrasoft E-Sign app. The user waits at step 3 for the tenant to sign on the device — move-in completes only after signature submission. The signed document is retrieved and can be printed or emailed post-signing. Supports resend if the document doesn't appear on the device.

Could Have · Synchronous
🔄

Post Move In Re-delivery

After any move-in, the lease can be regenerated and re-delivered using any supported method. This covers cases like an email that wasn't received, a customer requesting a print copy, or transitioning from Esign Pending back to an in-person sign. Status can be manually overridden by authorised users.

Must Have · Flexible
6Configuration Layer

What facilities can configure

Move-in behaviour is extensively configurable at the facility level — covering billing strategy, default documents, and charge categories — so different storage businesses can adapt it to their operational model.

Proration Settings Non-Anniversary Only

  • Enable / disable proration globally
  • Prorate First — bill prorated rent for first month
  • Prorate Second — full first month, prorated second
  • Prorate First Bill Second — prorate + bill full 2nd period upfront
  • Prorate First Bill Second After Days — day-threshold hybrid

Sync Billing Anniversary Only

  • Optionally sync rent assessment dates across a tenant's units
  • Checkbox shown only if ledger already has rented units
  • Toggling recalculates total payable with live snackbar feedback
  • Prorates the first period to the sync date

Charges Configuration

  • Setup fee — configurable charge category
  • Rental deposit — default charge category
  • Tenant deposit — default charge category
  • All charges individually waivable at move-in time

Documentation Defaults

  • Welcome letter — select template + auto-print toggle (default: true)
  • Demographics letter — letter + delivery method selection
  • Lease style — fallback if unit type has no configured lease
  • Unit type lease overrides facility-level lease style
7Outcomes

Key outcomes & contributions

Move In is the most critical user-facing workflow in the platform — every tenant relationship starts here. Building it required precision across billing logic, legal document handling, state management, and multi-mode UX.

01

Unified multi-mode onboarding — Move In, Estimate, and Reservation from one surface

Designed a single stateful 4-step flow that handles three fundamentally different modes simultaneously, with data-aware mode switching, cumulative alerts, and intelligent retention of compatible fields — preventing data loss while guiding users through state transitions.

02

Complex billing logic for anniversary and non-anniversary facilities

Implemented a proration engine supporting four distinct strategies plus sync-billing for anniversary facilities, with real-time bill-to-date recalculation and live total updates as users adjust dates, periods, or sync toggles during the move-in flow.

03

Three-path e-signature delivery with status-aware unit state management

Built lease delivery flows for print (synchronous, in-person), email e-sign (async with Esign Pending status and 7-day validity), and device sign (real-time, blocking until signed) — with post-move-in re-delivery and manual status override for authorised users.

04

Legally-compliant, template-driven lease and welcome letter generation

Built auto-populated legal lease documents and welcome letters with dynamic fields (tenant name, unit number, unit size, rate, dates, gate code) — generated and delivered at move-in time per facility-configured templates, with all output samples validated against real Catskill Self Storage documents.

05

7 access entry points with consistent context propagation

Move In is reachable from the Unit List, BEV, Edit Unit, View Tenant, Tenant List, Lead List, Edit Lead, and Quick Link — each access point pre-populating the relevant context (unit, tenant, lead) so the user lands at the right step with the right data already loaded.

06

Lead-to-tenant conversion built into the onboarding flow

Designed and built the lead conversion path directly within step 2 of Move In — pre-populating tenant fields from lead data, retaining compatible information across mode switches, and enabling the full tenant form inline without navigating away from the flow.
