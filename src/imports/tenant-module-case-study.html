01 — Background

What is this project?

TFW (The Facility Works) is a comprehensive self-storage facility management platform built at CodeParva. The Tenant Module sits at the core of the product — it is the primary interaction layer between facility staff and the customers (tenants) who rent storage units. This module spans the complete tenant lifecycle: from onboarding and identity collection, through payment and ledger management, to document generation, notifications, and eventual deactivation.

My contributions covered the design and engineering of multiple features and iterative versions within the Tenant Module — working across configuration systems, UX enhancements, payment integrations, and audit/compliance flows.

03 — Iterative Delivery

Feature versions & decisions

Each feature area was delivered in versioned increments — below is a representative view of the evolution of key features.

Tenant List Page 4 versions
v1 · Initial Build

Core list with search, filter & export

Delivered tenant columns (name, ledger, address, contact, balance), status and ledger-balance filters, name search, and PDF export. Established the foundational data model for the list page.

v2 · More Sorts

Extended sort capabilities

Added unit-number sort, ascending/descending balance sort, descending name sort, email sort, and primary phone sort — with graceful fallback ordering for tenants with missing data.

v3 · Name Representation

Last name first — platform-wide

Standardised name display to Last, First across 15+ surfaces: tenant list, unit list, lead list, reservations, waiting list, batch payments, late events, vehicles, pending tasks, and more.

v4 · Delinquency Indicators

Visual delinquency flagging

Highlighted balances in red for tenants who qualify for late/lockout events — even if the event hasn't been processed yet. Required migration logic to accurately carry forward delinquency status.

View Tenant 6 versions
v1 · Core Profile

Tenant profile, ledger summary & CRM controls

Built the full View Tenant screen including basic info, ledger summary (balance, auto-pay status, units, due dates), gate controls, alerts, unpaid bills, contact log, reservations, and website access controls.

v2 · Document Management

Separated saved documents from document generation

Split the documentation UX into: (1) Saved documents with filter, delivery status and re-delivery options, (2) Generate Letter redirect flow, and (3) Create Invoice with type/style/ledger selection.

v3 · Alert Integration

Feature-scoped tenant alerts with severity

Introduced configurable alerts (Warning, Error, Query, Information) with feature-level scoping — alerts could be set to show on Move In, View Tenant, and/or Payments. Supports activate/deactivate from the tenant profile.

v4 · File Upload

PDF document attachment per tenant

Enabled upload, view (in-app PDF viewer), download, and delete of arbitrary documents against a tenant — with metadata capture (file name, size, uploader, timestamp) and migration for legacy documents.

v5 · Unified Contact Log

Consolidated all communication in one timeline

Extended the contact log to capture prints, reprints, and generated documents across 14+ workflows (move-in, move-out, payments, late events, retail, transfer, vehicle, reports) alongside existing email/SMS/manual entries.

v6 · Deactivation Safety

Dependency checks before tenant deactivation

Implemented pre-deactivation validation covering 10 dependencies: ledger balance, pending invoices, active reservations, tenant deposits, gate codes, alerts, saved cards, vehicles, retail returns, and document delivery restrictions.

Edit Tenant 4 versions
v1 · Foundation

Section-aware tenant form

Implemented configurable section visibility (show always / on-demand / never), required/preferred/optional fields, permission-based access, ledger editing with payment method and autopay, section collapse/expand.

v2 · Optional Fields UX

Guided discovery for optional fields

Added contextual hints when all fields in a section are optional — prompting the user to enable "Show Optional Fields" before dismissing a section as empty. Applied to Show Always and On-Demand section types.

v3 · Autopay Reset Logic

Clear autopay when payment method changes

When a tenant's preferred payment method is changed (e.g. from credit card to cash), the autopay enrollment is automatically cleared — preventing stale autopay configs from applying to new payment methods.

v4 · Conditional Autopay

Autopay gated on card/ACH availability

Autopay enrollment is only enabled when the selected payment method supports it (credit card or ACH) — and requires valid card/bank details to be present before the option can be activated.

Ledger Review 2 versions
v1 · Core Ledger

Chronological transaction log

Delivered a full transaction timeline (transaction number, unit, dates, amounts, balance, description) with reversal/deletion — mandating comments for audit purposes. Oldest-to-newest order for easy balance verification.

v2 · Merchant Portal Integration

Real-time credit card reversals via Syrapay

Integrated with the Syrapay merchant portal to void credit card transactions in real time. Built smart refund eligibility logic (180-day blind refund window, partial vs. full refund rules) and robust success/failure notification flows.

04 — What was achieved

Key outcomes & contributions

⚙️

Deep configurability for multi-facility deployments

Built a layered configuration system (section visibility, field-level required/optional/preferred, tenant deposit, duplicate-check toggle) so each self-storage facility could tailor the tenant experience without custom code.

💳

Real-time payment integration with merchant portal

Replaced a manual dual-system refund process by integrating directly with Syrapay — enabling credit card reversals from within the platform, reducing staff effort and reconciliation errors.

📋

Unified audit trail across the tenant lifecycle

Consolidated emails, SMS, prints, document generation, and manual contact logs into a single chronological contact log — giving facility managers a complete history of every interaction with a tenant.

🔔

Automated tenant & facility notification system

Designed and implemented 8 tenant-facing notification templates and a real-time facility pop-up system for online events (move-ins, payments, reservation, profile changes) — delivered via the integrated SMS product.

🛡️

Safe tenant deactivation with dependency validation

Built a pre-deactivation checklist that checks for outstanding balance, pending invoices, reservations, gate codes, saved cards, vehicles, and more — preventing data integrity issues caused by premature deactivation.

🚩

Proactive delinquency visibility

Introduced real-time delinquency flagging on the tenant list page — highlighting tenants who qualify for late or lockout events even before those events are processed, helping facility staff prioritise collections.
