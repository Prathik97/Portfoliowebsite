01 · Background

What is the Unit Module?

The Unit Module is the operational backbone of TFW — it governs how physical storage units are represented, configured, rented, billed, and maintained across a facility. Spanning everything from an interactive Bird's Eye View canvas to automated rent assessment pipelines and mobile walk-around tooling, this module required deep product thinking across visual UX, financial logic, and system configuration.

02 · Feature Areas

Nine areas of the Unit Module

Each area was independently scoped and versioned. Together they form a complete lifecycle for how a self-storage facility creates, prices, monitors, and bills its physical units.

01

Bird's Eye View (BEV)

An interactive canvas showing the spatial layout of all units across levels. Supports zoom, unit status colour-coding, setup mode for repositioning units, and hover tooltips with live tenant data.

Automated and manual billing pipeline generating charges across all tenants or a single unit/ledger. Supports anniversary and non-anniversary facility modes, payment-triggered assessment, and concurrency locking to prevent double-billing.

03 · Iterative Delivery

Feature versions & decisions

Features were shipped in successive versions, each addressing real user feedback, migration challenges, or client requests. Below is the versioned evolution of the most complex feature areas.

🗺️
Bird's Eye View (BEV) 6 versions
v1.0Core Canvas

Interactive facility map with setup mode

Built the foundational BEV canvas: multi-level support, zoom (40–160%), unit placement with setup mode (create/edit/remove), colour-coded rental/availability/delinquency status, and click-through navigation to view tenant or move-in flows.

v2.0Unit Search

Search-to-scroll with double-stroke highlight

Added unit search that scrolls and auto-switches levels to the searched unit's position. Used a double-stroke visual highlight instead of colour (since unit colour is status-driven and user-configured) to clearly identify the result — requested by Fullsteam.

v3.0Placeholders

Non-unit elements on the canvas

Introduced placeholder entities (Office, Lift, Road, Others with custom labels) with configurable dimensions. Decided to use text labels over icons after review. Delete restricted to unsaved or placeholder-only selections — units excluded from bulk delete.

v4.0Colour Resolution

Simplified delinquency colour model

Resolved conflicting colour logic across lockout events, late events, and auction states. Final decision: gate lockout triggers full unit colour (red default); all other delinquency events follow standard corner-indicator behaviour. Auction integrated into future delinquency event scope.

v5.0Canvas Scroll Fix

Arrow-key navigation in setup mode

Resolved a spacebar-capture issue in setup mode (caused by TeamViewer input interception) by implementing arrow-key canvas panning. Verified coordinate migration integrity as part of the fix.

v6.0Hover Tooltip

Rich unit info on hover — without a click

Replaced the basic unit-number tooltip with a rich hover card showing: unit number, unit type, rate (standard if vacant / current if rented), next billing date, tenant name, ledger balance, last paid date and amount, and full status. Integrated with BEV search to show the same card at the scrolled-to unit.

⚙️
Unit Type Configuration 4 versions
v1.0Foundation

Full unit type config with rent slider

Built the core unit type setup: description, setup fee, deposit, charge category, rates by period, dimensions, category (Climate Controlled, Outdoor, Mobile, Automobile), overbook limit, and the rent slider — a dynamic pricing engine that adjusts move-in rates based on facility occupancy thresholds using flat or percentage calculations.

v2.0Deactivation

Safe unit type deactivation

Added eligibility-gated deactivation — a unit type can only be deactivated if no units are tagged to it. Deactivation triggers automatic cleanup of any rent adjustments linked to the unit type.

v3.0Bulk Update

Batch rate and fee updates across unit types

Enabled bulk editing of standard rate, web rate, deposit, setup fee, charge category, and lease style across multiple unit types and periods in one operation. Built transactional failure handling — partial failures roll back the entire batch to prevent inconsistent state. Lease and website rates separated as distinct action items for clarity.

v4.0Warehouse Category

Warehouse as a supported unit category

Added Warehouse as a system-level unit category following a client request (Herlihy, FID: 29421). Scoped future extensibility via an "Others / Custom" category path for any further bespoke requests.

📈
Rent Adjustment 4 versions
v1.0Wizard Base

Bulk rent adjustment with multi-criteria filtering

Built the rent adjustment wizard allowing facility managers to select units by rental status, unit type, period, last-adjustment date, and occupancy tenure. Supports 8 adjustment calculation modes (flat, percentage of current/standard, higher/lower of flat vs percent, increase to standard), configurable recurrence, and tenant notification via print or email — with template saving for reuse.

v2.0Terminology

UX clarity and status column improvements

Introduced a status column (Scheduled / Complete / Stopped), reorganised the affected units grid, simplified tenant-based filter labels ("Tenants who have not been adjusted before", "Tenants who have rented over"), and clarified recurrence UI with explicit period selectors — based on user feedback sessions with Tom.

v3.0Increase %

Show percentage impact at the confirmation step

Added a summary percentage showing how much the scheduled adjustment increases or decreases rates overall — calculated as the delta between sum of all scheduled rates vs. all current rates for affected units, expressed as a percentage change. Helps managers validate the impact before committing.

v4.0Single Unit Adjustment

Per-unit adjustment from rate history

Introduced single-unit rent adjustment accessible directly from the Edit Unit rate history panel — showing full tenant context (current rate, standard rate, move-in date, last increase date, next assessment date). Supports all 8 calculation modes, rounding options, configurable notification timing with alternate contact, and conflict detection for existing scheduled adjustments. Requested by Tama Toledo (FID: 15458).

✏️
Edit Unit 2 versions
v1.0Core Editor

Full unit editor with rate history lifecycle

Built the complete Edit Unit screen: rental status transitions (rented ↔ vacant with appropriate data clearing and defaults), current rate and next assessment date management, insurance configuration, invoicing settings (enable/disable, invoice-to date, always-generate toggle), rate history with Moved In / Pending / Applied / Waived / Reversed / Moved Out statuses, and maintenance event tracking per unit.

v2.0Migration Rate Fix

Correct current rate mapping from v10 migration

Addressed a migration bug where the next rent assessment rate was incorrectly used as the current rate during v10→v11 migration — causing incorrect billing when recurring charges, insurance, or promo plans were present. Fixed by migrating current charges from v10 as independent line items (recurring charges, insurance, promo) rather than collapsing them into a single rate figure.

🔄
Recurring Charges 2 versions
v1.0Core Feature

Add-on charges billed every rent assessment cycle

Built recurring charge creation on units — supporting a description, start/end date, charge category, flat or percentage type, and rounding. Charges auto-generate on each rent assessment cycle. Users can manually discontinue from the next period. Migrated from v10 with automatic RC 1/RC 2 naming for entries missing titles.

v2.0Migration Name Fix

Auto-naming blank recurring charge entries

Fixed a migration edge case (reported at Harrison Storage, FID: 1838) where recurring charges migrated from v10 without a title were left unnamed. Added logic to auto-assign RC 1, RC 2, etc., per unit to ensure the charges are identifiable in the UI post-migration.

🚶
Walk Around 1 version
v1.0Initial Build

Mobile-optimised unit patrol and overlock management

Designed for staff walking the physical facility on a mobile or tablet device. Features: configurable unit order (5 system presets — last name alpha, unit number asc/desc, address, zip + 1 custom sort), bulk overlock/unlock status updates with partial-save on failure, and access to rate history, tenant info, move-in and move-out flows per unit. Migrated v10 custom sort orders per unit from tblSortOrder/tblUnit_tblSortOrder. Requested by Tim/Tom.

04 · Outcomes

Key outcomes & contributions

Across the Unit Module, the work spanned complex UX, financial engine design, data migration strategy, and multi-facility configurability — resulting in a production-grade module used across diverse storage facility types.

01

Visual facility management via interactive canvas

Built a zoomable, multi-level BEV canvas that gives facility managers a live spatial overview of unit status, delinquency, and gate lockout — with rich hover tooltips and direct navigation — replacing guesswork with at-a-glance insight.

02

Dynamic pricing engine with occupancy-based rent slider

Designed and implemented the rent slider — a real-time move-in pricing engine that automatically discounts or premiums rates based on facility occupancy thresholds, configurable by unit type with flat or percentage calculation modes.

03

Complete rate history audit trail per unit

Delivered a full rate history ledger per unit tracking every rate event — move in, applied adjustments, pending schedules, waivers, reversals, and move out — with attribution to users and linkage to lease numbers, enabling transparent financial audit.

04

Bulk rent adjustment with 8 calculation modes

Built a flexible bulk rent adjustment wizard supporting 8 increase/decrease calculation strategies across unit subsets, with configurable recurrence, multi-channel tenant notification, warning thresholds, and reusable templates for routine rate cycles.

05

Resilient data migration from v10 to v11

Identified and resolved critical migration bugs — including incorrect current rate mapping caused by recurring charges, insurance, and promo plans being collapsed into a single figure — ensuring billing continuity post-migration for all facilities.

06

Mobile-first walk-around tooling for field staff

Designed a dedicated mobile experience for facility staff to patrol units, update overlock statuses in bulk, and access tenant/unit info on the go — with configurable sort orders and graceful partial-save failure handling.
